FlowRun 개인정보 처리방침

최종 수정일: 2026년 5월 9일 · 적용 버전: v3.3.3 이상

FlowRun(이하 "본 확장")은 Google Labs Flow에서의 AI 이미지·영상 배치 생성 자동화를 제공하는 Chrome 확장 프로그램입니다. 본 방침은 본 확장이 처리하는 정보의 종류, 목적, 저장 위치, 사용자 권리를 설명합니다.

1. 수집·처리하는 개인정보 항목

항목출처수집 시점저장 위치
이메일 주소 (email) Google OAuth (chrome.identity) 최초 로그인 시 1회 Firebase Firestore + chrome.storage.local
표시 이름 (displayName) Google OAuth 최초 로그인 시 1회 Firebase Firestore + chrome.storage.local
프로필 사진 URL (photoURL) Google OAuth 최초 로그인 시 1회 Firebase Firestore + chrome.storage.local
Google 계정 고유 ID (uid) Firebase Auth 최초 로그인 시 생성 Firebase Firestore + chrome.storage.local
최근 활동 시각 (lastActiveAt) 본 확장 자체 기록 로그인·상태 확인 시 갱신 Firebase Firestore
사용자 승인 상태 (status) 관리자가 부여 (pending/approved) 최초 등록 시 pending, 관리자 승인 시 approved Firebase Firestore
Firebase 인증 토큰 (idToken, refreshToken) Firebase Auth 로그인 시 발급 chrome.storage.local (브라우저 내)

수집하지 않는 정보: 비밀번호, 결제 정보, 위치, IP 주소, 웹 방문 기록, 키 입력 로그, 마우스 위치, 다른 사이트의 콘텐츠, Google Flow 외부의 어떤 데이터도 수집하지 않습니다.

2. 처리 목적

  • 사용자 인증 및 승인 관리: 본 확장은 관리자 승인 기반 베타 서비스입니다. Google 계정으로 로그인한 사용자의 승인 상태(승인/대기)를 확인해 기능 접근을 제어합니다.
  • 중복 가입 방지 및 사용자 식별: uid를 키로 하나의 Google 계정당 하나의 사용자 레코드만 생성합니다.
  • 본 확장 운영자에게 사용자 식별 표시: 관리자가 승인 요청을 검토할 때 본인 확인용으로 email/displayName 노출.

사용하지 않는 목적: 광고, 마케팅, 통계 분석, 외부 판매, 신용도 판단, 머신러닝 학습 데이터.

3. 저장 위치 및 제3자

  • Firebase Firestore (Google Cloud) — 프로젝트 ID: flowrun-kr. Google이 관리하는 인프라이며, Google의 개인정보 처리방침이 함께 적용됩니다.
  • chrome.storage.local — 사용자 본인 브라우저 내부에만 저장. 외부 송신 없음.
  • 제3자 제공 없음: 본 확장은 Firebase(Google) 외 어떠한 제3자에게도 사용자 데이터를 판매·전송·공유하지 않습니다.

4. 보존 및 삭제

  • 보존 기간: 사용자가 삭제 요청을 하기 전까지 또는 본 확장이 종료될 때까지 Firestore에 보관됩니다.
  • 로그아웃 시: chrome.storage.local의 토큰·캐시 정보는 즉시 삭제되지만, Firestore 사용자 레코드는 별도 삭제 요청 전까지 유지됩니다.
  • 확장 제거 시: chrome.storage.local 데이터는 Chrome에 의해 자동 삭제됩니다. Firestore 레코드는 자동 삭제되지 않으므로 제거를 원하시면 아래 연락처로 요청해 주세요.

5. 사용자의 권리

본 확장 사용자는 언제든 다음을 요청할 수 있습니다:

  • 본인의 Firestore 레코드 열람 (저장된 email, displayName 등 확인)
  • 본인의 Firestore 레코드 삭제 (계정 정보 영구 삭제)
  • 처리 정지 요청 (서비스 이용 중단)

요청은 아래 GitHub Issues 또는 이메일을 통해 접수합니다. 신원 확인 후 7영업일 이내에 처리합니다.

7. 외부 서비스

  • Google Labs Flow (labs.google/fx) — 사용자가 직접 상호작용하는 대상. 프롬프트, 생성 결과는 Google Flow의 처리방침이 적용됩니다.
  • Firebase / Firestore (Google Cloud) — 사용자 인증·승인 상태 저장. Google의 처리방침이 함께 적용됩니다.

8. 미성년자

본 확장은 만 14세 미만 아동의 데이터를 의도적으로 수집하지 않습니다. Google 계정 정책에 따라 가입 가능한 연령 이상에서만 사용 가능합니다.

9. 방침 변경

본 방침이 변경될 경우 이 페이지의 "최종 수정일"을 갱신합니다. 중대한 변경 시 본 확장 내 공지 시스템을 통해 안내합니다.

FlowRun Privacy Policy (English)

Last updated: May 9, 2026 · Applies to: v3.3.3+

FlowRun is a Chrome extension that automates AI image and video batch generation on Google Labs Flow. This policy describes the personal data we process, why, where it is stored, and your rights.

1. Personal Data We Process

ItemSourceWhen collectedStored at
EmailGoogle OAuthFirst sign-in (once)Firebase Firestore + chrome.storage.local
Display nameGoogle OAuthFirst sign-in (once)Firebase Firestore + chrome.storage.local
Profile photo URLGoogle OAuthFirst sign-in (once)Firebase Firestore + chrome.storage.local
Google account UIDFirebase AuthCreated on first sign-inFirebase Firestore + chrome.storage.local
Last active timestampExtension itselfUpdated on sign-in/status checkFirebase Firestore
Approval statusSet by adminInitial: pending; admin: approvedFirebase Firestore
Firebase ID/refresh tokensFirebase AuthOn sign-inchrome.storage.local (browser only)

NOT collected: passwords, payment info, location, IP address, browsing history, keystroke logs, mouse position, content from other sites, or any data outside Google Flow.

2. Purposes

  • User authentication and approval management: FlowRun is an admin-approved beta service. We check the approval status (pending/approved) of users signed in with their Google account to gate feature access.
  • Duplicate-account prevention and identification: The UID acts as a primary key — one Google account = one user record.
  • Identification for the extension operator: Email/displayName are shown to the admin during approval review only.

NOT used for: advertising, marketing, statistical analytics, third-party sale, credit scoring, or ML training.

3. Storage and Third Parties

  • Firebase Firestore (Google Cloud) — project ID flowrun-kr. Managed infrastructure by Google; Google's privacy policy also applies.
  • chrome.storage.local — stored only inside your own browser; not transmitted externally.
  • No third-party sharing: FlowRun does not sell, share, or transfer user data to any party other than Firebase (Google) infrastructure.

4. Retention and Deletion

  • Retention period: Records remain in Firestore until you request deletion or the extension is discontinued.
  • On sign-out: chrome.storage.local tokens/cache are immediately cleared; Firestore record persists until separate deletion request.
  • On extension uninstall: chrome.storage.local is auto-deleted by Chrome. Firestore record is NOT auto-deleted; contact us to delete it.

5. Your Rights

You may request at any time:

  • Access to your Firestore record (view stored email, displayName, etc.)
  • Deletion of your Firestore record (permanent removal)
  • Suspension of processing (stop using the service)

Submit requests via the contact methods below. We respond within 7 business days after identity verification.

6. Permissions Used

See the Korean section above (Section 6) for the full list. Each permission is used solely for the stated purpose; debugger attaches only to https://labs.google/* tabs and detaches immediately after a single trusted click.

7. External Services

  • Google Labs Flow (labs.google/fx) — your direct interaction target; Flow's privacy policy applies to prompts and generated content.
  • Firebase / Firestore (Google Cloud) — stores authentication and approval status; Google's privacy policy applies.

8. Children

FlowRun does not knowingly collect data from children under the age permitted by Google account policy.

9. Policy Updates

We update the "Last updated" date on changes. Material changes are announced via the in-extension notice system.
